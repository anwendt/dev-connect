# dev-connect v0.2.0 Release Notes

## Overview

`v0.2.0` is the first minor release after the initial `v0.1.x` implementation
series. It focuses on operational readiness, documentation accuracy, Windows
client usability, VS Code Remote SSH interoperability, and development server
onboarding.

## Highlights

- Added a full development server onboarding guide for Ubuntu 24.04 LTS
  targets.
- Documented Linux user creation, SSH public key installation, host key
  collection, GitOps host key inventory updates, gateway deployment, and client
  smoke tests.
- Added remote VS Code Server proxy setup documentation for extensions that run
  on the remote host, including GitHub Copilot scenarios.
- Clarified `kubectl` discovery across `--kubectl-path`,
  `DEV_CONNECT_KUBECTL_PATH`, `clusters.<name>.kubectlPath`, bundled
  `kubectl`, `PATH`, and OS default paths.
- Updated API documentation for current commands, flags, and configuration
  top-level fields.
- Corrected Windows YAML examples to use single-quoted paths.
- Updated the Helm chart `appVersion` metadata to `0.2.0`.

## Client Notes

- Windows clients can use a colocated `kubectl.exe` next to
  `dev-connect.exe`.
- Proxy overrides under `clusters.<name>.proxy` apply only to local `kubectl`
  child processes.
- Remote VS Code Server proxy settings are configured separately through
  `vscode.remoteSetup`.
- `ssh.manageUserConfig: true` remains the recommended mode when using the
  normal local VS Code profile with browser-based authentication flows.

## Gateway Notes

- No gateway behavior change is required for this release.
- Existing gateway deployments can continue to run.
- The Helm chart metadata now reflects application version `0.2.0`.

## Upgrade Notes

- Existing `dev-connect/v1` client configuration remains compatible.
- No Kubernetes manifest migration is required.
- No host key inventory migration is required.
- Review client configuration if Windows workstations rely on proxy or
  `kubectl` discovery behavior.

## Validation

The release should be validated with:

```text
make release-check
make build-all VERSION=v0.2.0
make package-windows-bundle VERSION=v0.2.0
make helm-package
make checksums
```
